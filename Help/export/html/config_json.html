Configuração de Arquivo JSON

O arquivo JSON contém as definições e configurações necessárias para a exportação dos produtos cartográficos, como escala, simbologia, camadas e layout. Abaixo está um guia de como configurar corretamente este arquivo.

1. Estrutura do Arquivo JSON

Os itens destacados em AZUL são obrigatórios. Itens opcionais estão em VERDE.

Atributo Atributo no JSON Características / Exemplos
Tipo de Produto tipo_produto Preencher com "Carta Topográfica".
Edição do Produto edicao_produto Preencher com a Edição do Produto.
Licenciamento Produto licenciamento_produto "CC-BY-SA 4.0" ou "CC-BY-NC-SA 4.0".
Índice de Nomenclatura inom Ex: SF-22-Y-C-I-1. No caso de cartas Não-SCN, esse atributo não é utilizado.
Escala escala Indica a escala do produto. Exemplo: 25000, 50000.
Centro da Carta center Localização do centro do produto. Com latitude e longitude em graus decimais.
Nome nome Nome da carta. Ex: "Rio Laranjaí".
Cabeçalho cabecalho Caminho do arquivo QPT a ser inserido na parte superior esquerda da carta.
Créditos projeto Caminho do arquivo QPT a ser inserido na parte inferior esquerda da carta.
Direitos de Reprodução direitos_reproducao Caminho do arquivo QPT a ser inserido na parte inferior direita da carta.
Acesso à Informação acesso_informacao Caminho do arquivo QPT a ser inserido na parte inferior direita da carta.
Território Internacional territorio_internacional Define se o produto engloba área internacional. Valores possíveis: true ou false.
Acesso Restrito acesso_restrito Define se o produto é de acesso restrito. Valores: true ou false.
MDE para Diagrama de Elevação mde_diagrama_elevacao Defina o caminho para o MDE e seu EPSG.
Caminho da MDE mde_diagrama_elevacao/caminho_mde Caminho absoluto para o arquivo MDE.
Conexão com Banco de Dados banco Informações da conexão com banco de dados. Exemplo: servidor, porta, nome.
Fases de Produção fases Lista com as fases de produção, como "Imageamento", "Aquisição Vetorial".
Executantes fases/executantes Lista de executantes com atributos como nome e ano de execução.
PEC Planimétrico info_tecnica/pec_planimetrico Ex: "A".
PEC Altimétrico info_tecnica/pec_altimetrico Ex: "A".
Dados de Terceiros info_tecnica/dados_terceiros Informações de terceiros, como FUNAI e IBAMA.
Data Criação info_tecnica/data_criacao Preencher Data.
Origem de Dados Altimétricos info_tecnica/origem_dados_altimetricos Ex: GLO-30 com correção do efeito do dossel vegetativo.
Datum Vertical info_tecnica/datum_vertical Informações do Datum Vertical.

OBSERVAÇÃO: Se o atributo projeto não for incluído no arquivo JSON, o plugin buscará automaticamente o arquivo creditsDSG.qpt, que já está presente no diretório do plugin por padrão. Caso seja necessário definir um arquivo QPT específico para os créditos ou outras informações, inclua o caminho no atributo projeto.
