Descrição do Plugin

Esta ferramenta permite a exportação de Cartas em formato PDF e GeoTIFF, com base em configurações estabelecidas em arquivos JSON e layouts definidos em arquivos QPT.

Seleção do Tipo de Produto

Na caixa Selecione o tipo de produto, você pode escolher dentre diferentes tipos de produtos geoespaciais. Cada tipo de produto pode alterar a forma como os dados são representados, processados e exportados. Alguns exemplos de produtos disponíveis são:

  • Carta Topográfica: Representação cartográfica convencional usada para a visualização de elementos geográficos em escalas como 1:25.000, 1:50.000, 1:100.000, ou 1:250.000. Utilizada para aplicações gerais de geoinformação, permitindo a visualização de relevo, hidrografia, vegetação e infraestrutura.

  • Carta Ortoimagem: Combina dados vetoriais com imagens ortorretificadas (ortofotos), permitindo uma visualização precisa e detalhada da superfície terrestre. Este produto é ideal para análises que exigem imagens reais e detalhamento espacial, como monitoramento ambiental ou planejamento urbano.

  • Carta Ortoimagem Militar: Variante militar da Carta Ortoimagem, que inclui dados sensíveis e simbologias personalizadas voltadas para operações estratégicas e de defesa.
  • Carta Topográfica Militar: Adaptada para fins militares, essa carta inclui dados táticos e camadas adicionais de segurança e acessibilidade, utilizadas em operações militares ou missões especiais.

Esses são apenas exemplos, e novos produtos podem ser adicionados ou modificados de acordo com as necessidades do projeto.

Configuração dos Arquivos JSON

Os arquivos JSON usados no plugin servem para carregar e configurar parâmetros pré-definidos para a execução das ferramentas de edição. Eles atuam como arquivos de configuração, permitindo que o sistema leia as definições necessárias sem a necessidade de configurá-las manualmente a cada execução.

Esses arquivos JSON não têm a função de criar elementos dentro do projeto. Eles apenas carregam dados e configurações previamente estabelecidos, garantindo a consistência e o funcionamento adequado ao executar a ferramenta.

Certifique-se de que os arquivos JSON estão corretamente estruturados e que contêm todas as informações necessárias para o uso correto da ferramenta, como parâmetros e outros ajustes técnicos.

Parâmetros de Conexão

No campo Parâmetros de conexão, o usuário deve fornecer as informações necessárias para que o plugin possa se conectar aos bancos de dados geoespaciais referenciados no arquivo JSON. Isso garante que as camadas de dados sejam carregadas corretamente durante a exportação.

Configuração e Edição do Arquivo QPT

O arquivo QPT (template de layout) é utilizado para definir a estrutura dos produtos gerados pelo plugin. Cada produto, tem um layout específico que é gerado de forma sistemática, dependendo do tipo de produto e da legenda dinâmica.

Não é recomendável que o operador faça alterações no arquivo QPT dos produtos, pois isso pode desconfigurar a exportação e comprometer o funcionamento do plugin. Alterações não autorizadas podem quebrar o processo automatizado de criação e exportação dos layouts.

A única edição permitida no QPT é na seção de creditsDSG, onde o operador pode ajustar os créditos e informações relacionadas. No entanto, é importante seguir o padrão estabelecido para o tamanho e posicionamento do creditsDSG, uma vez que qualquer alteração significativa pode impactar no posicionamento de outros elementos no rodapé (rodamão) da carta.

Certifique-se de seguir os padrões recomendados para garantir a correta exportação e apresentação dos produtos.
