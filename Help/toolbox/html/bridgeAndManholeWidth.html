Descrição:

O algoritmo define a largura de representação das pontes, viadutos e outras feições que compõem a camada de elemento viário, levando em consideração, inclusive, as sobreposições com as vias de deslocamento.

Parâmetros:

• Selecionar a camada de elemento viário (Ponto) → Indicar a camada que contém os elementos viários com geometria do tipo ponto. Há a possibilidade de executar apenas os elementos selecionados. • Selecionar o atributo de largura → Indicar o atributo da camada acima que contém a informação da largura da simbologia. • Selecionar a camada de elemento viário (Linha) → Indicar a camada que contém os elementos viários com geometria do tipo linha. Há a possibilidade de executar apenas os elementos selecionados. • Selecionar o atributo de largura → Indicar o atributo da camada acima que contém a informação da largura da simbologia. • Selecionar camada de rodovia → Indicar a camada que contém as vias de deslocamento.

Exemplo de uso:

Modelagem EDGV3.0_TOPO na escala de 1:25.000 (EPSG):4674

• Selecionar a camada de elemento viário (Ponto) → infra_elemento_viario_p • Selecionar o atributo de largura → largura_simbologia • Selecionar a camada de elemento viário (Linha) → infra_elemento_viario_l • Selecionar o atributo de largura → largura_simbologia • Selecionar camada de rodovia → infra_via_deslocamento_l

Aplicação:

Atributo 'largura_simbologia' preenchido adequadamente.
